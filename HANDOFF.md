# Snap Workbench — handoff brief for Claude Code

## What this is
A single-file Marvel Snap deck-building PWA, self-hosted on GitHub Pages, with a
GitHub Action that keeps the card database current daily. Built iteratively in a
Claude chat; moving to CC for ongoing work.

## Repo layout
- `index.html` — the entire app (HTML + CSS + JS, card DB snapshot embedded).
  Runs both as a Claude artifact and self-hosted; storage auto-detects
  (Claude `window.storage` -> browser `localStorage` -> in-memory).
- `cards.json` — generated daily by the Action. Contract:
  `{ updated: "YYYY-MM-DD", cards: [{ n, d, c, p, a, s, f?, i? }] }`
  (name, CardDefId, cost, power, ability, series, flavor, art URL).
  `index.html` `loadSiteData()` fetches this at boot with cache:no-store.
- `update-cards.mjs` — the pipeline. Two sources:
  1. marvelsnapzone getinfo feed: card data, but `status` stays "unreleased"
     for weeks after event releases, and tokens/summons are also
     type=Character + unreleased.
  2. snap.fan/news/schedule/: forward-looking release dates; card links use
     real CardDefIds as URL slugs. Entries roll off within days of release.
  Logic: released -> in; SEED list or previous cards.json membership -> in
  (persistence: once published, never drops out); else schedule date decides;
  unscheduled unreleased = token = silently skipped. Sanity floor: refuses to
  write under 200 cards. Name filters: " - " (Team Clash dupes),
  endsWith(" Champion") (Sanctum Showdown dupes).
  NOTE: strings are backtick-only to survive iPhone paste; CC edits files
  directly, so feel free to normalize.
- `.github/workflows/update-cards.yml` — dual cron 19:01 + 20:01 UTC so one
  run always lands 3:01 PM Eastern across DST, plus workflow_dispatch.
  Deliberately zero quote characters (mobile-paste survival); safe to normalize.
- `manifest.webmanifest`, `icon-*.png` — PWA identity for Add to Home Screen.
- Unused hooks in index.html: `REFRESH_URL` / `COACH_URL` constants for an
  optional Cloudflare Worker (one-tap refresh + AI coach proxy off-Claude).
  Worker source exists in the kit zip from the chat if ever wanted.

## Deck code format (verified against the live game)
base64(JSON) of `{ Name, Cards: [{ CardDefId }] }`. The game ACCEPTS codes
generated by `exportCode()` (confirmed in-game). The game's own clipboard
output may be unpadded base64 and/or wrapped in surrounding text.

## OPEN BUG: importing Snap's clipboard into the app fails
- Where: `parseCode()` in index.html (used by `openImport()`).
- Already handled: unpadded base64, URL-safe chars, longest-run extraction
  from surrounding text, invisible unicode stripping.
- Leading hypothesis: newer Snap copies a share URL with percent-encoded
  base64 (`%2B`/`%2F`/`%3D`), which fragments the run-matcher. Fix sketch:
  try `decodeURIComponent` on the paste (guarded try/catch) before extraction;
  also consider extracting a `deck=` query param if present.
- First step: capture the exact clipboard payload from the game into Notes
  and characterize it before coding.

## Tests
`snap-workbench.test.js` (jsdom harness, ~40 assertions) exercises boot,
grid, deck zone, filters, sort, sheet, codes round-trip, stub imports,
DB updater, storage shim, AI mock. Run:
  npm i jsdom && node snap-workbench.test.js
It targets `index.html` in the same directory. Extend it with a regression
test using the REAL captured Snap clipboard payload once obtained.

## Known environment facts
- marvelsnapzone card pages return no usable date text to server-side fetch
  (scraping them failed 80/80) — don't retry that path; the schedule works.
- Art hotlinks from marvelsnapzone CDN render fine on GitHub Pages.
- The mid-2025 DB snapshot embedded in index.html is only a fallback;
  cards.json overrides it at boot.
